## 1. 产品概述

### 1.1 产品背景

随着内容创作生态的蓬勃发展，个人博客作为独立内容平台重新获得关注。相较于微信公众号、微博等平台，独立博客在内容所有权、SEO 优化、读者关系管理等方面具有独特优势。本产品旨在为内容创作者提供一套功能完善、体验流畅的博客系统，帮助其建立个人品牌，沉淀知识资产。

### 1.2 产品愿景

> 成为每位知识创作者的"数字家园"——让创作简单、让读者留下、让内容长久流传。

### 1.3 成功指标（North Star Metric）

- **主要指标**：月活跃博主数（MAU-Writer）
- **辅助指标**：
  - 平均文章发布频率（篇/月/人）
  - 读者平均阅读时长（分钟）
  - 文章自然搜索流量占比（SEO 健康度）

## 2. 产品定位

### 2.1 目标市场

| 维度 | 描述 |
|------|------|
| 市场规模 | 中国独立博客/知识创作者约 500 万人，活跃度高的约 80 万人 |
| 竞品 | WordPress、Hexo、Ghost、掘金、语雀 |
| 市场机会 | 轻量化、开箱即用、国内访问稳定、中文友好 |

### 2.2 产品价值主张

**"5分钟开始写作，一键发布，永久留存"**

- 🚀 **极简上手**：无需技术背景，注册即写
- ✍️ **创作自由**：Markdown + 富文本双模式
- 🔍 **SEO 友好**：自动生成结构化数据，提升自然搜索排名
- 🏠 **内容自主**：数据完全归作者，支持一键导出

### 2.3 差异化定位

```
                    重 功 能
                      │
         WordPress ●  │  ● 企业级 CMS
                      │
  复杂 ───────────────┼─────────────── 简 单
                      │
         Hexo ●       │  ● 本产品（★）
                      │
                    轻 量 化
```

**本产品定位**：面向个人创作者，轻量、简洁、强 SEO，介于 Hexo（静态、无后台）与 WordPress（功能庞大、学习成本高）之间的黄金地带。

## 5. MVP 范围

### 5.1 MVP 定义

MVP（最小可行产品）目标：**让一个技术/知识博主能够在本系统注册、写文章、发布并分享给读者阅读**，完整跑通"创作-发布-阅读"核心链路。

### 5.2 MVP 功能集

#### ✅ 纳入 MVP 的功能（全部为 P0 项）

| # | 功能 | 纳入理由 |
|---|------|---------|
| 1 | 邮箱注册/登录 | 用户体系基础，无法绕过 |
| 2 | 个人主页设置（头像/简介） | 博客品牌最低要求 |
| 3 | Markdown 编辑器（含代码高亮） | 核心创作工具，缺失则无法写作 |
| 4 | 图片上传 | 文章内容完整性必需 |
| 5 | 草稿自动保存 | 防止内容丢失，用户体验底线 |
| 6 | 文章发布/取消发布 | 发布能力核心 |
| 7 | 文章分类与标签 | 内容最基础的组织方式 |
| 8 | 文章列表页 + 详情页 | 读者核心阅读路径 |
| 9 | 分类/标签归档页 | 内容发现最低要求 |
| 10 | 移动端响应式设计 | 超过 60% 读者来自移动端 |
| 11 | 文章阅读量统计 | 作者需要最基本反馈 |
| 12 | 自动生成 sitemap.xml | SEO 收录最低要求 |
| 13 | RSS Feed | 内容分发/订阅基础能力 |
| 14 | 文章管理后台（列表/删除） | 内容管理基础 |

#### ❌ 暂不纳入 MVP 的功能及理由

| 功能 | 暂缓理由 |
|------|---------|
| 富文本编辑器 | Markdown 已满足核心用户需求，双编辑器复杂度高，MVP 期专注 Markdown |
| 评论功能 | 初期用户量小，互动需求不迫切；可接入第三方评论系统（如 Disqus）作临时方案 |
| 定时发布 | 频次低，MVP 期手动发布已满足需求 |
| Google Analytics 集成 | 可通过 GA 独立配置替代，不影响核心流程 |
| 自定义域名 | 技术复杂度高，MVP 提供默认子域名即可 |
| 主题切换 | 先固定一套精美主题，避免分散设计资源 |
| 文章点赞/分享卡片 | 锦上添花功能，可在 v1.1 加入 |
| 数据导出 | 用户量小阶段风险低，v1.1 加入 |

### 5.3 MVP 成功标准

- 博主可在 5 分钟内完成注册并发布第一篇文章
- 文章详情页 Lighthouse 性能评分 ≥ 85
- 首屏加载时间 < 2 秒（4G 网络）
- 0 个 P0 级 Bug 上线

## 7. 功能详细说明（PRD 正文）

### 7.1 用户体系

#### 7.1.1 注册与登录

**功能描述**  
用户通过邮箱+密码方式完成注册，系统发送验证邮件激活账号。登录支持"记住我"功能。

**用户故事**

```
As a 访客
I want to 通过邮箱快速注册账号
So that 我可以开始创建自己的博客

验收标准：
✓ 邮箱格式校验，重复邮箱提示"该邮箱已注册"
✓ 密码强度要求：至少8位，含字母和数字
✓ 注册后发送激活邮件，24小时内有效
✓ 激活后自动登录并跳转到个人主页设置页
✓ 登录失败5次后，锁定15分钟
```

**UI 设计要点**
- 注册/登录共用一个页面，通过 Tab 切换
- 表单字段：邮箱输入框（带格式实时校验）、密码框（带显隐切换）、确认密码（仅注册态）
- 登录页底部提供"忘记密码"链接，触发邮件重置流程
- 错误提示用红色文字，显示在对应输入框下方

---

#### 7.1.2 个人主页设置

**功能描述**  
博主设置博客基本信息，包括博客名称、个人简介、头像、社交链接等，构建博主的公开形象页。

**用户故事**

```
As a 已注册博主
I want to 设置我的博客名称和个人介绍
So that 读者能了解我是谁，以及我的博客定位是什么

验收标准：
✓ 必填项：博客名称（5-50字）
✓ 选填项：个人简介（不超过200字）、头像上传（支持 JPG/PNG，≤ 2MB）
✓ 社交链接：最多支持 5 个，字段含平台类型下拉（GitHub/Twitter/微博/知乎/自定义）+ URL 输入
✓ 博客默认 URL：{domain}/{username}，用户名注册后不可更改
✓ 保存成功后跳转到博客主页预览
```

### 7.2 内容创作

#### 7.2.1 文章编辑器（Markdown 模式）

**功能描述**  
提供左右分栏的 Markdown 编辑体验：左侧为 Markdown 源码编辑区，右侧为实时渲染预览区。

**用户故事**

```
As a 技术博主
I want to 用 Markdown 编写文章并实时看到渲染效果
So that 我可以高效地撰写技术文章，而无需关心排版细节

验收标准：
✓ 编辑器支持标准 Markdown 语法（CommonMark 规范）
✓ 代码块支持语言指定，渲染后呈现语法高亮（使用 Prism.js 或 highlight.js）
✓ 支持的代码语言：JavaScript/TypeScript/Python/Java/Go/Shell/SQL/HTML/CSS 等主流语言
✓ 图片支持拖拽上传，自动插入 Markdown 图片语法
✓ 支持 GFM 扩展：表格、任务列表、删除线、脚注
✓ 编辑区与预览区滚动同步
✓ 支持全屏编辑模式
✓ 每 30 秒自动保存为草稿，状态栏显示"上次保存时间"
```

**UI 设计要点**
- 顶部工具栏：加粗 / 斜体 / 链接 / 图片 / 代码 / 代码块 / 引用 / 有序列表 / 无序列表 / 分割线
- 右上角操作区：发布按钮（主按钮，蓝色）、保存草稿（次要按钮）、预览（全屏）
- 字数统计：底部状态栏显示"字数：1234 | 阅读时间：约 5 分钟"
- 编辑区背景色：亮色模式 `#FAFAFA`，暗色模式 `#1E1E1E`

---

#### 7.2.2 图片上传

**功能描述**  
支持在编辑器内上传图片，图片存储至云端对象存储，返回可访问 URL。

**用户故事**

```
As a 博主
I want to 在文章中上传本地图片
So that 我的文章可以包含丰富的视觉内容，而不依赖外部图床

验收标准：
✓ 支持拖拽到编辑区直接上传
✓ 支持点击工具栏图片按钮选择文件上传
✓ 支持格式：JPG / PNG / GIF / WebP
✓ 单张图片大小限制：≤ 5MB（免费用户），≤ 20MB（付费用户）
✓ 上传中显示进度条
✓ 上传成功后自动插入 Markdown 图片语法：`![alt](url)`
✓ 上传失败提示具体原因（格式不支持/超出大小限制/网络错误）
```

#### 7.2.3 草稿自动保存

**功能描述**  
编辑器内容每 30 秒自动保存到服务端草稿，防止意外关闭导致内容丢失。

**用户故事**

```
As a 正在写作的博主
I want to 我的文章内容可以自动保存
So that 即使意外关闭浏览器，也不会丢失已写的内容

验收标准：
✓ 自动保存间隔：30 秒
✓ 保存状态显示在编辑器顶部："正在保存..." → "已保存 19:38"
✓ 网络断开时：本地 localStorage 兜底存储，恢复联网后自动同步
✓ 重新进入草稿时，自动加载最新草稿内容，顶部提示"已恢复上次草稿"
✓ 同一篇文章只保留最新一个草稿版本（不做版本历史，v2.0 考虑）
```

### 7.3 内容发布

#### 7.3.1 文章发布流程

**功能描述**  
博主点击"发布"按钮后，弹出发布配置面板，完成元信息配置后正式发布。

**用户故事**

```
As a 博主
I want to 在发布前配置文章的分类、标签和摘要
So that 读者能更好地发现和理解我的文章

验收标准：
✓ 点击"发布"弹出侧边栏面板（不离开编辑器）
✓ 发布配置项：
  - 文章标题（必填，自动从正文第一行H1提取，可修改）
  - 文章摘要（选填，不填则截取正文前150字）
  - 封面图（选填，支持上传或从正文图片中选择）
  - 分类（单选，从已有分类选择或新建，必填）
  - 标签（多选，最多 5 个）
  - 发布状态：立即发布 / 保存为草稿
✓ 点击"确认发布"后，页面跳转到已发布文章的阅读页
✓ 文章 URL 格式：/{username}/{article-slug}
  - article-slug 默认由标题自动生成（中文转拼音，空格转连字符）
  - 支持手动修改 slug（仅在发布前可修改）
```

#### 7.3.2 文章分类与标签

**功能描述**  
分类是文章的主分组维度（树形一级），标签是多维度的轻量标注。

| 属性 | 分类（Category） | 标签（Tag） |
|------|----------------|------------|
| 层级 | 一级平铺 | 无层级 |
| 数量限制 | 博主最多 20 个分类 | 系统共享，每篇文章最多 5 个 |
| 文章归属 | 每篇文章归属 1 个分类 | 每篇文章最多 5 个标签 |
| URL 形式 | `/category/{slug}` | `/tag/{slug}` |

### 7.6 SEO 模块

#### 7.6.1 Sitemap 自动生成

**功能描述**  
系统自动维护 `sitemap.xml`，每次文章发布/更新/删除后触发异步更新。

**Sitemap 结构要求**

```xml
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://{domain}/{username}</loc>           <!-- 博客主页 -->
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://{domain}/{username}/{slug}</loc>    <!-- 文章页 -->
    <lastmod>2026-06-27</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
  </url>
  <!-- 分类页、标签页 priority: 0.6 -->
</urlset>
```

## 8. 非功能需求

### 8.1 性能需求

| 指标 | 要求 |
|------|------|
| 文章详情页首屏加载 | < 2s（4G 网络，1MB 以内页面资源） |
| 编辑器响应延迟 | 键入后渲染延迟 < 100ms |
| API 响应时间（P95） | < 500ms |
| 系统可用性 | ≥ 99.5%（月度） |

### 8.2 安全需求

| 安全项 | 措施 |
|--------|------|
| XSS 防护 | Markdown 渲染后对 HTML 进行白名单过滤（DOMPurify） |
| CSRF 防护 | 所有修改性 API 携带 CSRF Token |
| 密码存储 | bcrypt 加盐哈希，不明文存储 |
| 图片上传 | 服务端校验 MIME 类型，禁止可执行文件伪装 |
| HTTPS | 强制 HTTPS，HTTP 自动重定向 |

### 8.3 兼容性需求

| 维度 | 要求 |
|------|------|
| 浏览器 | Chrome 90+ / Firefox 88+ / Safari 14+ / Edge 90+ |
| 移动端 | iOS 14+ / Android 10+ |
| 屏幕适配 | 最小适配宽度：375px（iPhone SE） |

### 8.4 可访问性（A11Y）

- 图片必须含 `alt` 文字
- 颜色对比度满足 WCAG 2.1 AA 级别（4.5:1）
- 键盘可导航（Tab 焦点流清晰）
